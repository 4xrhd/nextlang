# NetXLang Project Report

## Implemented Features
- **Compiler Pipeline**: Successfully implemented Lexer, Parser, Semantic Analyzer, and Interpreter in C++17.
- **Language Support**: All 25 keywords implemented (some with simplified simulation logic).
- **Network Simulation**: Virtual devices, links, IPs, ports, and packet routing logic are fully functional.
- **Security Features**: Firewall (`ProtirodhDao`) and Packet Drop (`HariyeFelo`) simulation implemented.
- **Control Flow**: `Jodi` (if), `Nahole` (else), and `GhuroChol` (loop) work as expected.
- **Testing**: Unit tests for Lexer, Parser, and Semantic Analyzer. Integration tests using example `.nxl` files.

## Known Limitations
- **OpenSSL Integration**: While OpenSSL was detected on the system, headers were not found in standard paths. Encryption (`GoponKoro`) and Decryption (`UghatKoro`) are currently simulated using string labels. In a production environment, linking against the OpenSSL library path would enable real AES-256.
- **Expression Evaluation**: Current expression evaluation is simplified to string representations for constants and variables.
